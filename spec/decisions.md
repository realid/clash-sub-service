# 决策记录

## 退出码

- `0`：成功
- `2`：参数或配置错误，可立即修复
- `3`：运行时错误，例如网络、IO、校验失败

当前阶段未新增其他退出码。

## 工程结构

- 新实现统一放在 `clash_sub_service/`
- Python 源码直接放在 `src/` 下
- 采用 `src/` 布局

## 配置策略

- 示例配置文件放在 `config.example.yaml`
- 真实订阅链接不得入库
- 真实订阅链接通过本机 `config.yaml` 管理，不使用环境变量覆盖

## 日志策略

- 默认日志输出到 `stdout`
- 同时支持文件日志
- `stdout` 使用 `colorlog`
- 文件日志不带颜色
- URL query 中的敏感字段必须脱敏
